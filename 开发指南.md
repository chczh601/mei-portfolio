# Mei个人网站 - 开发指南

## 1. 项目概述

本项目是一个面向数字经济专业学生Mei的个人博客/作品集网站，用于展示个人信息、技能、项目成果和联系方式。网站采用现代化设计，包含特色粒子效果和响应式布局。

## 2. 技术栈选择

### 2.1 前端技术
- **HTML5**: 语义化页面结构，构建网站基础框架
  - 理由：标准Web技术，兼容性好，易于维护
- **CSS3**: 响应式样式设计，实现现代化视觉效果
  - 理由：支持Flexbox和Grid布局，实现复杂响应式设计；支持CSS动画和过渡效果
- **JavaScript (ES6+)**: 实现交互功能和动画效果
  - 理由：原生JavaScript，无需额外依赖，运行效率高
- **Canvas API**: 实现粒子效果
  - 理由：直接操作像素，实现高性能的粒子动画
- **Font Awesome 6.4.0**: 提供图标支持
  - 理由：丰富的图标库，易于使用，提升视觉效果
- **Inter Font**: 网站主要字体
  - 理由：现代无衬线字体，可读性好，设计简洁

### 2.2 后端技术
- **静态网站**: 无需复杂后端
  - 理由：降低维护成本，提高运行效率，适合个人展示网站

### 2.3 开发工具
- **Git**: 版本控制
  - 理由：管理代码变更，便于协作和回滚
- **GitHub**: 代码托管和项目部署
  - 理由：全球最大的代码托管平台，提供免费的静态网站部署服务
- **GitHub Pages**: 静态网站部署平台
  - 理由：与GitHub集成，自动部署，无需额外配置

## 3. 本地运行步骤

### 3.1 克隆仓库

```bash
git clone https://github.com/chczh601/mei-portfolio.git
```

### 3.2 进入项目目录

```bash
cd mei-portfolio
```

### 3.3 运行本地服务器

由于这是一个静态网站，你可以使用任何静态文件服务器来运行它。以下是几种常用方法：

#### 方法1：使用Python内置服务器（推荐）

```bash
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000
```

#### 方法2：使用Node.js的http-server

```bash
# 全局安装http-server（如果未安装）
npm install -g http-server

# 运行服务器
http-server -p 8000
```

#### 方法3：使用VS Code的Live Server扩展

1. 在VS Code中打开项目目录
2. 安装Live Server扩展
3. 点击右下角的"Go Live"按钮

### 3.4 访问网站

打开浏览器访问以下地址：

```
http://localhost:8000
```

## 4. 目录结构说明

```
mei-portfolio/
├── css/               # CSS样式文件
│   └── style.css      # 主样式文件
├── images/            # 图片资源
│   ├── Mei.jpg        # 个人照片
│   └── [其他照片]     # 相册照片
├── js/                # JavaScript文件
│   ├── main.js        # 主要交互逻辑
│   └── particles.js   # 粒子效果实现
├── index.html         # 主页面
├── README.md          # 项目说明文档
├── pro.md             # 产品需求文档
├── LLM协助全过程记录.md  # 开发记录文档
└── 开发指南.md         # 开发指南（当前文件）
```

### 4.1 目录详细说明

- **css/style.css**: 包含网站所有样式，包括响应式设计、动画效果和主题色设置
- **images/**: 存放网站所有图片资源，包括个人照片和相册照片
- **js/main.js**: 实现网站主要交互功能，如导航、表单验证和动态内容加载
- **js/particles.js**: 实现Canvas粒子效果，包括鼠标引力和弹性连线功能
- **index.html**: 网站主页面，包含所有内容区块和页面结构
- **README.md**: 项目基本说明，用于GitHub仓库展示
- **pro.md**: 产品需求文档，详细描述网站功能和设计要求
- **LLM协助全过程记录.md**: 记录开发过程中的协作历史
- **开发指南.md**: 本文件，提供开发环境搭建和项目结构说明

## 5. 内容更新指南

### 5.1 更新个人信息

1. 打开 `index.html` 文件
2. 找到 `<section id="about">` 区块
3. 修改相关文本内容
4. 如需更换个人照片，将新照片命名为 `Mei.jpg` 并替换 `images/Mei.jpg`

### 5.2 更新技能列表

1. 打开 `index.html` 文件
2. 找到 `<section id="skills">` 区块
3. 修改或添加技能卡片

### 5.3 更新项目展示

1. 打开 `index.html` 文件
2. 找到 `<section id="projects">` 区块
3. 修改或添加项目卡片
4. 更新项目描述和链接

### 5.4 更新博客内容

1. 打开 `index.html` 文件
2. 找到 `<section id="blog">` 区块
3. 修改或添加博客卡片

### 5.5 更新联系信息

1. 打开 `index.html` 文件
2. 找到 `<section id="contact">` 区块
3. 修改联系信息和社交媒体链接

### 5.6 更新相册内容

1. 将新照片添加到 `images/` 目录
2. 照片将自动显示在相册中，使用文件名作为描述

## 6. 样式定制

### 6.1 修改主题色

1. 打开 `css/style.css` 文件
2. 找到 `:root` 变量定义或主色调设置（#B6DDDC）
3. 修改颜色值

### 6.2 修改粒子效果

1. 打开 `js/particles.js` 文件
2. 修改粒子系统参数，如粒子数量、颜色、大小和动画速度

### 6.3 添加新的动画效果

1. 打开 `css/style.css` 文件
2. 添加新的CSS动画定义
3. 在HTML元素中应用动画类

## 7. 部署指南

### 7.1 GitHub Pages部署

1. 确保所有更改已提交到GitHub仓库的main分支
2. 打开GitHub仓库页面
3. 点击 "Settings" -> "Pages"
4. 确保源设置为 "Deploy from a branch"，分支选择 "main"
5. 点击 "Save"
6. 等待部署完成（通常需要1-2分钟）
7. 访问 `https://chczh601.github.io/mei-portfolio/` 查看部署结果

### 7.2 解决部署版本不更新问题

如果部署后看不到最新更改，可以尝试以下方法：

1. 修改 `README.md` 文件（如添加一个空格）
2. 提交更改到GitHub
3. 这将触发GitHub Pages重新构建和部署

## 8. 故障排除

### 8.1 粒子效果不显示

- 检查浏览器控制台是否有JavaScript错误
- 确保 `js/particles.js` 文件正确加载
- 检查Canvas元素是否存在于HTML中

### 8.2 图片不显示

- 检查图片路径是否正确
- 确保图片文件存在于 `images/` 目录中
- 检查图片文件名是否大小写敏感

### 8.3 样式显示异常

- 检查浏览器控制台是否有CSS错误
- 确保 `css/style.css` 文件正确加载
- 检查CSS选择器是否正确

### 8.4 表单提交不工作

- 检查浏览器控制台是否有JavaScript错误
- 确保表单验证逻辑正确
- 注意：由于是静态网站，表单提交需要后端支持才能实际发送邮件
